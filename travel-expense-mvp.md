# 朋友结伴出游记账管理网站 MVP

## 目标

做一个面向朋友结伴出游的旅行账本网站。用户登录后可以创建出游记录，在旅行过程中持续记录账单，旅行结束后自动计算总金额、每人应分摊金额、个人已付/应付/应收/应补金额，并输出账单明细和结算建议。

## 核心用户场景

1. 用户注册并登录网站。
2. 用户创建一次出游记录，例如“杭州三日游”。
3. 用户添加同行成员，可以是已注册用户，也可以是临时昵称成员。
4. 旅行过程中记录每笔消费，包括金额、付款人、参与分摊成员、分类、日期和备注。
5. 出游详情页展示总花费、成员分摊结果、账单明细和结算建议。
6. 旅行结束后，系统给出谁应该给谁转多少钱，并支持标记结清。

## MVP 功能范围

### 用户与权限

- 用户注册
- 用户登录
- 用户退出
- 用户只能查看自己创建或参与的出游记录
- 未登录用户不能访问出游、账单和结算页面

### 我的出游

- 查看自己的出游列表
- 创建出游
- 编辑出游基础信息
- 查看出游详情

出游基础信息包括：

- 名称
- 开始日期
- 结束日期
- 地点
- 币种
- 备注

### 出游成员

- 添加同行成员
- 成员可以关联注册用户
- 成员也可以只是临时昵称
- 查看成员列表
- 删除未产生账单关联的成员

第一版建议保留“临时昵称成员”，不要强制所有朋友注册。

### 账单记录

- 添加账单
- 编辑账单
- 删除账单
- 查看账单明细列表

单笔账单字段包括：

- 所属出游
- 金额
- 付款人
- 消费分类
- 消费日期
- 说明
- 参与分摊成员
- 每个成员分摊金额

第一版默认使用平均分摊，同时为后续自定义分摊金额保留数据结构。

### 汇总统计

出游详情页展示：

- 总花费
- 人均花费
- 每人已付款金额
- 每人应分摊金额
- 每人净额
- 每人应收或应补金额

计算公式：

```text
净额 = 已付款总额 - 应分摊总额
```

- 净额大于 0：该成员应收款
- 净额小于 0：该成员应补款
- 净额等于 0：该成员已平账

### 结算建议

系统根据成员净额生成转账建议。

算法思路：

1. 将净额小于 0 的成员作为付款方。
2. 将净额大于 0 的成员作为收款方。
3. 按金额从大到小匹配付款方和收款方。
4. 每次生成一条转账记录，直到所有成员净额归零。

结算建议示例：

```text
张三 给 李四 转 120.50 元
王五 给 李四 转 80.00 元
```

### 结清状态

- 查看结算建议
- 标记某条结算已完成
- 查看已结清 / 未结清状态

## 推荐页面

1. 登录页
2. 注册页
3. 我的出游列表页
4. 创建 / 编辑出游页
5. 出游详情页
6. 添加 / 编辑账单页
7. 结算页

出游详情页建议包含四个区域：

- 概览
- 成员
- 账单明细
- 结算

## 推荐后端模型

### User

使用 Django 自带用户模型。

### Trip

出游记录。

核心字段：

- owner
- title
- location
- start_date
- end_date
- currency
- note
- created_at
- updated_at

### TripMember

出游成员。

核心字段：

- trip
- user，可为空
- display_name
- created_at

说明：`user` 可为空，用于支持未注册朋友以昵称参与记账。

### Expense

账单。

核心字段：

- trip
- payer
- amount
- category
- spent_at
- description
- created_by
- created_at
- updated_at

金额必须使用 Decimal 类型，不使用 float。

### ExpenseSplit

账单分摊明细。

核心字段：

- expense
- member
- amount

### Settlement

结算记录，可选但建议 MVP 保留。

核心字段：

- trip
- from_member
- to_member
- amount
- is_paid
- paid_at
- created_at

## 推荐 API

第一版可以使用 Django REST Framework。

建议接口：

- `POST /api/auth/register/`
- `POST /api/auth/login/`
- `POST /api/auth/logout/`
- `GET /api/trips/`
- `POST /api/trips/`
- `GET /api/trips/{id}/`
- `PATCH /api/trips/{id}/`
- `DELETE /api/trips/{id}/`
- `GET /api/trips/{id}/members/`
- `POST /api/trips/{id}/members/`
- `GET /api/trips/{id}/expenses/`
- `POST /api/trips/{id}/expenses/`
- `PATCH /api/expenses/{id}/`
- `DELETE /api/expenses/{id}/`
- `GET /api/trips/{id}/summary/`
- `GET /api/trips/{id}/settlements/`
- `PATCH /api/settlements/{id}/`

## 权限规则

- 出游创建者默认拥有管理权限。
- 出游参与成员可以查看该出游。
- 非出游成员不能查看出游、账单、汇总和结算。
- 账单创建、修改和删除第一版可以只允许出游创建者操作。
- 后续可以扩展为所有出游成员都能记账。

## 技术建议

当前项目已经是 Vue + Django：

- 后端继续使用 Django。
- API 层建议引入 Django REST Framework。
- 前端继续使用 Vue + Vite。
- 金额字段使用 Decimal。
- 日期使用本地时区。
- 用户认证第一版可以使用 session 或 token，后续再扩展 JWT。
- 后台管理继续使用 Django admin + simple-ui。

## 第一版验收标准

1. 用户可以注册、登录和退出。
2. 登录用户可以创建出游记录。
3. 登录用户可以看到自己的出游列表。
4. 出游中可以添加至少 3 个同行成员。
5. 可以添加多笔账单，并选择付款人和参与分摊人。
6. 系统可以自动计算总金额和人均金额。
7. 系统可以展示每个成员的已付款、应分摊、应收和应补金额。
8. 系统可以展示完整账单明细。
9. 系统可以输出结算建议。
10. 非相关用户不能访问该出游记录。

## 暂不纳入 MVP

- 好友系统
- 群聊
- 图片票据识别
- 多币种实时汇率
- 微信/支付宝真实支付
- 复杂审批流
- 多角色团队权限
- 公开分享页
